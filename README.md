# Streamlit Workshop

A hands-on workshop for exploring [Streamlit](https://streamlit.io/) features using open data sources. Participants build interactive data applications using Python, Plotly, Polars, and DuckDB across three real-world use cases.

## Use Cases

### Titanic

Explore the classic Titanic passenger dataset. Analyse survival rates by class, gender, and age using interactive charts. This is the simplest use case and requires no data preparation — the CSV is included in the repository.

### Land Registry

Analyse UK property sales data from the HM Land Registry Price Paid dataset, combined with ONS postcode geolocation data. Explore price trends by property type, postcode area, and time period with interactive maps and charts.

**Note:** This use case downloads approximately 3 GB of data during preparation.

### World Bank

Visualise global development indicators from the World Bank Open Data API. Compare countries over time across metrics such as GDP per capita, life expectancy, and population — recreating the classic "Wealth & Health of Nations" visualisation.

## Prerequisites

- [Git](https://git-scm.com/)
- One of the following:
  - [VS Code](https://code.visualstudio.com/) + [Docker](https://www.docker.com/) (Dev Container)
  - [uv](https://docs.astral.sh/uv/) (local install)
  - Python 3.13 + pip (local install)

## Installation

Clone the repository:

```bash
git clone https://github.com/Smartitect/streamlit-workshop.git
cd streamlit-workshop
```

### Option 1: Dev Container (VS Code + Docker)

This is the recommended approach. It provides a fully configured environment with all dependencies pre-installed.

1. Install [VS Code](https://code.visualstudio.com/) and [Docker Desktop](https://www.docker.com/products/docker-desktop/)
2. Install the [Dev Containers extension](https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-containers) in VS Code
3. Open the cloned repository folder in VS Code
4. When prompted, click **"Reopen in Container"** (or use the Command Palette: `Dev Containers: Reopen in Container`)
5. Wait for the container to build — this will automatically install `uv` and run `uv sync` to set up the Python environment

The container is ready when the terminal is available.

### Option 2: uv (local)

[uv](https://docs.astral.sh/uv/) manages both the Python version and dependencies for you.

1. [Install uv](https://docs.astral.sh/uv/getting-started/installation/)
2. Sync the project:

```bash
uv sync
```

This will automatically download Python 3.13 (if not already installed) and create a virtual environment with all dependencies.

### Option 3: pip (local)

Requires Python 3.13 already installed on your system.

1. Create and activate a virtual environment:

```bash
python3.13 -m venv .venv
source .venv/bin/activate  # On Windows: .venv\Scripts\activate
```

2. Install dependencies:

```bash
pip install -r requirements.txt
```

## Getting Started

### Titanic

No data preparation required — run the notebook directly:

1. Open `notebooks/titanic/titanic_data_exploration.ipynb`
2. Run all cells to explore the data

To run the Streamlit app:

```bash
cd app
streamlit run titanic_app.py
```

### Land Registry

1. Open and run `notebooks/land_registry/prepare_land_registry_data.ipynb` to download and prepare the data (~3 GB download)
2. Open `notebooks/land_registry/query_land_registry_database.ipynb` to explore the data interactively

To run the Streamlit app:

```bash
cd app
streamlit run land_registry_app.py
```

### World Bank

1. Open and run `notebooks/world_bank/prepare_world_bank_data.ipynb` to download indicator data from the World Bank API
2. Open `notebooks/world_bank/explore_world_bank_data.ipynb` to explore and visualise the data
